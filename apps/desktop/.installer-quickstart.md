Build installers for Portal Fusion in 3 easy steps!

## Step 1: Install Dependencies

```bash
cd apps/desktop
npm install
```

## Step 2: Build Installers

```bash
# Build for your current platform
npm run build

# Or build for specific platforms:
npm run build:win    # Windows (NSIS, MSI, APPX)
npm run build:mac    # macOS (DMG, PKG, ZIP)
npm run build:linux  # Linux (AppImage, deb, rpm, snap)
```

## Step 3: Find Your Installers

```bash
ls -la dist/
```

Installers will be in the `dist/` directory!

---

## What Gets Built

### Windows
- **Portal Fusion-1.0.0-win-x64.exe** - NSIS installer (recommended)
- **Portal Fusion-1.0.0-win-x64.msi** - MSI installer (enterprise)
- **Portal Fusion-1.0.0-win-x64.appx** - Microsoft Store package

### macOS
- **Portal Fusion-1.0.0-mac-x64.dmg** - DMG installer for Intel Macs
- **Portal Fusion-1.0.0-mac-arm64.dmg** - DMG installer for Apple Silicon
- **Portal Fusion-1.0.0-mac-x64.pkg** - PKG installer for Intel Macs
- **Portal Fusion-1.0.0-mac-arm64.pkg** - PKG installer for Apple Silicon

### Linux
- **Portal Fusion-1.0.0-linux-x86_64.AppImage** - Universal package
- **Portal Fusion-1.0.0-linux-amd64.deb** - Debian/Ubuntu package
- **Portal Fusion-1.0.0-linux-x86_64.rpm** - RedHat/Fedora package
- **Portal Fusion-1.0.0-linux-amd64.snap** - Snap package

## Common Tasks

### Clean build artifacts
```bash
npm run clean
```

### Check what will be built
```bash
npm run installer:help
```

### Build without creating installer (testing)
```bash
npm run build:quick
```

### Update version
Edit `package.json`:
```json
{
  "version": "1.0.0"  // Change this
}
```
